# 《去不同的站》（暂定名）

这是一个十年前温州背景的高中校园言情故事。林知夏和陈既白分别是高二（3）班与高二（7）班的英语课代表，因为交听写本、取试卷、核对名单、帮郑老师传话而频繁在英语办公室和走廊碰面。两个人都成绩好、可靠、受欢迎，也都背着家庭期待和“不能出错”的压力。故事不写高调表白，也不靠强情节推进，而是把喜欢藏在纸质试卷、公交站、晚自习、短信句号、排名表和一次次错开的目光里。关系从陌生注意、互相试探、暗中偏心，到被流言和成绩压力推远；高考后，他们承认彼此重要，却没有给未来一个封闭答案。结尾停在杭州与上海、公交站与车站之间，保留开放分别的余味。

## 项目结构

- `chapters/`：正文与单章计划，当前完成 38 章。
- `planning/`：故事圣经、全书大纲、阶段规划等。
- `reviews/`：连续性日志、阶段复盘、AI 味与结构检查记录。
- `skills/`：本项目使用的本地写作与审校 skills。

## 写作工作流

本项目不是直接一口气生成全文，而是按章迭代：

1. 先维护故事圣经，确定人物、关系模型、校园空间、尺度边界和结尾方向。
2. 再写全书大纲与阶段规划，把关系推进拆成可检查的小事件。
3. 每章先做章节计划，再由干净上下文的子 agent 起草正文。
4. 主线程验收章节，检查连续性、人物状态、AI 味、时代感和章节钩子。
5. 更新连续性日志，记录时间线、关系状态、伏笔和风险。
6. 每 5-10 章做阶段复盘，修正节奏、重复桥段和后续优先级。
7. 通过验收后再落盘，进入下一章。

## 本地 Skills

项目通过本地 skills 拆分职责，常用角色包括：

- `showrunner`：维护全书方向、关系阶段和章节推进。
- `campus-romance-chapter-drafter`：按章节计划起草正文，保证每章有具体校园事件。
- `campus-romance-continuity-editor`：检查时间线、人物状态、关系进度、伏笔和设定一致性。
- `style-polisher`：压低解释性、模板化和“AI 味”表达，让语言更像中文校园叙事。
- `campus-romance-arc-reviewer`：每个阶段复盘节奏、情绪变化、重复桥段和开放结尾方向。

这些角色配合使用，目标是让正文逐章积累，而不是用一次性生成替代结构、审校和修订。

## 当前产物

- 正文：38 章。
- 体量：约 12.8 万中文字符。
- 结尾：开放式分别。林知夏去杭州，陈既白去上海；两人确认过彼此的重要性，但没有承诺未来。

## 阅读顺序

1. 先读 `planning/story_bible.md`，了解设定、人物和尺度边界。
2. 再读 `planning/outline.md` 与 `planning/final_arc_plan.md`，看全书结构和终章收束。
3. 正文按 `chapters/chapter_01.md` 到 `chapters/chapter_38.md` 顺序阅读。
4. 如需检查创作过程，可对照 `reviews/continuity_log.md` 和 `reviews/arc_reviews.md`。

## 维护说明

- 不建议继续补多年后番外，避免削弱开放结尾。
- 若修改正文，同步更新连续性日志和相关阶段复盘。
- 修改人物设定、大学去向、关键物件时，先回看故事圣经和终章复盘。
- 新章节或修订仍应保持克制校园言情的尺度：用动作、物件、沉默和错过推进情绪。
